## Overview

This register tracks implementation and evidence for the SWIFT Customer Security Controls Framework (CSCF).

**Source:** `Source Docs/CSCF_v2026_20250701_compared_to_v2025.pdf`

**Note on Mandatory vs Advisory:** This template marks controls as **Advisory** when the control ID includes the `A` suffix (for example, `2.4A`). The CSCF also includes architecture-specific advisory applicability in its summary table; always refer to the source document for the authoritative applicability.

### Status legend

| Status | Meaning |
|---|---|
| Implemented | Control is implemented and evidenced |
| In Progress | Implementation/evidence work is underway |
| Planned | Work is scheduled but not started |
| Not Applicable | Not applicable to selected architecture/scope (justify) |

### Evidence expectations (minimum)

For each control, keep:

- **Policy/procedure reference(s)** (what requires it)
- **Technical/config evidence** (how it’s enforced)
- **Operational evidence** (that it runs, e.g., reports/logs/screenshots)
- **Owner and review cadence**

Use `templates/control-assessment-record.md` for consistent evidence capture.
